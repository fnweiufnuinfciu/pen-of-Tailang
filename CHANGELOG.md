# 更新日志

## v1.3.0 — 动态状态锚点 (2026-06-23)

### 新增
- **StateAnchor 状态管理类**：轻量级 JSON 结构追踪角色状态、位置、物品、禁止事件
- **关系状态追踪**：正则自动检测告白/热恋/分手/已婚等关系阶段变化，防止下一章关系倒退
- **人物名称锁定**：从角色卡自动提取并锁定核心人物名，防止 AI 擅自改名
- **锚点双重注入**：system prompt 末尾 + user prompt 开头双重强化，`!!! 核心设定 !!!` 标记
- **禁止项检测与自动重试**：若生成内容违反硬性禁止列表，自动触发重写（最多 3 次）
- **mini_goal 小目标**：支持传入"本章必须达成"的临时检查点（如"本章结尾出现狼嚎"）
- 锚点 UI 面板：手动设置初始状态 + 实时显示当前锚点

### 改进
- 上下文窗口精简：仅保留最近 2 章全文 + 紧凑锚点串，不再塞入全量历史
- buildUserPrompt 独立函数：增强提示词移入 user 消息（而非 system），规避 DeepSeek 审核
- parseOutline：丢弃导言文本（标题行、写作说明），只解析真正的章节标记

### 修复
- 章节解析不再多生成一章（序言被正确丢弃）
- regenerateChapter 同步使用锚点机制

## v1.2.0 — 单章重生成与服务器升级 (2026-06-17)

### 新增
- **单章重生成**：每章完成后的 🔄 按钮，单独重写不重新生成
- **角色卡可视化编辑**：JSON 编辑 / 表单编辑双模式切换，自动双向同步
- **开始新作**按钮：一键清空大纲、角色卡、缓存、日志
- 章节列表状态指示器：⏳ 待生成 / ✍️ 生成中 / ✅ 完成 / ❌ 失败
- server.py 时间戳日志（写入 server.log）
- start.bat：崩溃自动重启

### 改进
- 发送到生成器后自动解析章节，无需手动点击解析
- 修改大纲或角色卡时自动作废旧的解析缓存
- 导出 TXT 包含完整章节标题
- 章节列表改为数字索引引用（不再嵌入大段 HTML 属性）

### 修复
- localStorage 空值导致 `Cannot read properties of null` 崩溃
- use_proxy 复选框残留导致 JS 报错
- 章节列表显示重复与缺失章节号

## v1.1.0 — 两段式生成与规划器审核绕过 (2026-06-16)

### 新增
- **故事规划器**：输入需求 → AI 生成章节大纲 + 角色卡
- **一键发送到生成器**：大纲和角色卡自动填充并跳转
- 大纲/角色卡独立复制按钮
- **增强细节提示词**复选框（专用提示词注入）
- 每章最低字数配置

### 改进
- 合并规划器和生成器为单页双标签（墨韵金影主题）
- 连贯性保证：上一章结尾原文注入 + 关系状态追踪
- 大纲解析支持行内 `第X章` 标记（不再要求行首）
- 第X卷/第X节 自动过滤

### 修复
- 章节标题解析混乱（标题截取 20 字）
- 大纲中角色 JSON 混入章节显示
- 两步确认流程：先解析章节列表 → 确认 → 再生成
