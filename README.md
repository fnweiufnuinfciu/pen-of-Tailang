# 太郎的笔

一个自托管的 AI 长篇连小说写作工具。从故事规划、角色设定到逐章生成，一站式完成创作。

## 特性

- **故事规划** — 输入需求，AI 自动生成章节大纲 + 角色卡
- **逐章生成** — 两步操作（解析 → 确认 → 生成），实时进度 + 单章重写
- **动态状态锚点** — 自动追踪人物关系、角色名、身体状态、位置、物品，防止情节矛盾与角色名漂移
- **连贯性保证** — 上一章结尾原文注入 + 关系阶段锁定 + AI 自拟每章标题
- **角色卡编辑** — JSON 编辑器 + 可视化表单双模式
- **增强写作模式** — 可选的细节增强提示词，支持私密部位描写
- **一键导出 TXT** — 所有章节拼接下载
- **纯浏览器存储** — 配置、大纲、角色卡自动保存到 localStorage
- **零外部依赖** — Python 标准库即可运行

## 快速开始

### 前置要求

- Python 3.8+
- DeepSeek、OpenAI 或任何 OpenAI 兼容 API 的 Key

### 启动

```bash
python server.py
```

浏览器打开 `http://127.0.0.1:8765`

Windows 用户可直接双击 `start.bat`（崩溃自动重启）。

### 使用流程

```
故事规划 → 输入剧情框架 → 生成大纲 + 角色卡
    │
    ▼
发送到生成器 → 自动解析章节 → 确认章节目录
    │
    ▼
开始生成 → 逐章写作 → 实时进度 + 单章可重写
    │
    ▼
导出 TXT → 下载全文
```

## 大纲格式

支持 `第X章`、`## 标题`、`第一章` 等格式，章节标题可在行首或行中：

```
第1章 初见
新学期排座位，男女主成为同桌...

第2章 半块橡皮
她把橡皮掰成两半...
```

若大纲中未显式写标题，AI 生成时会自动拟题。

## 状态锚点

生成参数区提供"状态锚点"输入框，可手动设定初始状态。生成过程中系统会自动更新。

格式示例：

```
[状态]男主左臂骨折|内力三成 [位置]山洞 [物品]断剑|药瓶 [禁]用左拳 [目标]本章听到狼嚎
```

锚点自动追踪四项：

| 追踪项 | 说明 | 检测方式 |
|--------|------|----------|
| 人物关系 | 初识→暧昧→热恋→已婚 | 正则匹配"告白""在一起""结婚"等 |
| 人物名字 | 从角色卡自动提取 | 防止 AI 擅自改名 |
| 身体状态 | 受伤/昏迷/怀孕等 | 正则匹配 |
| 位置与物品 | 地点变更 + 物品获得/丢失 | 正则匹配 |

## 配置项

| 配置 | 说明 | 默认值 |
|------|------|--------|
| API Key | LLM API 密钥 | — |
| Base URL | API 端点 | `https://api.deepseek.com/v1` |
| 模型 | 模型名称 | `deepseek-chat` |
| Temperature | 创造性 (0–2) | 0.9 |
| 最大 Token | 每章最大输出 | 8000 |
| 每章最低字数 | AI 写作字数目标 | 2000 |
